## Typography (preserved from existing Vercel build)

**Display face: Outfit (free, Google Fonts).** Already loaded on the existing site. Geometric sans with strong character — distinct from the default-Inter SaaS template look. Use for all headings, hero display copy, wordmark, door headlines, metric numerals.

**Body face: Inter.** Already loaded on the existing site. Workhorse body face. Universal readability.

**Mono accent: JetBrains Mono.** Added for metric labels, small caps audience identifiers ("FOR FOUNDERS"), and any code/file-path references in case studies. Real mono — not a fallback to system mono.

**Why this pairing works:** Outfit + Inter is a real design choice (geometric display + neutral body), not a default. The risk-flag of "Inter alone = AI slop" is defused by Outfit doing all the heavy display work. Inter only carries body, where it's appropriate and unobtrusive.

**Type scale (rem-based, 1rem = 16px):**

| Role | Family | Size | Weight | Line height | Tracking |
|------|--------|------|--------|-------------|----------|
| Display (hero) | Outfit | 4.5rem (72px) → 6rem (96px) at ≥1280px | 700 | 1.0 | -0.02em |
| H1 (audience door headline) | Outfit | 2.25rem (36px) | 600 | 1.15 | -0.015em |
| H2 section heads | Outfit | 2rem (32px) | 600 | 1.2 | -0.01em |
| Body large | Inter | 1.25rem (20px) | 400 | 1.5 | 0 |
| Body | Inter | 1rem (16px) | 400 | 1.55 | 0 |
| Eyebrow / small caps | JetBrains Mono | 0.75rem (12px) | 500 | 1.4 | 0.12em uppercase |
| Metric numerals | Outfit | 3rem (48px) | 700, tabular-nums | 1.0 | -0.02em |
| Wordmark | Outfit | 1.5rem (24px) | 700 | 1.0 | -0.03em |

**Base body: 16px.** Anything smaller fails the readability bar.

**Font loading:** `next/font` self-hosted with `display: swap`. Subset to Latin only. (Currently loaded via Google Fonts CDN in the wireframe — production should self-host via `next/font/google`.)

## Color Palette (extracted from existing Vercel build)

| Token | Value | Usage | Source frequency in CSS |
|-------|-------|-------|-------------------------|
| `--color-bg` | `#FFFFFF` | Page background | High |
| `--color-bg-alt` | `#F8F9FA` | Card surfaces, light panels | 6 |
| `--color-fg` | `#1A1A2E` (near-black, slight blue undertone) | Headings, body text | 12 |
| `--color-fg-muted` | `#5A6478` (warm slate grey) | Secondary text, metadata | NEW (added — palette had no muted-grey token, this picks one that harmonizes with `#1A1A2E`) |
| `--color-border` | `#E2E5EB` (light cool grey) | Card borders, dividers | NEW (added — palette had no border token) |
| `--color-accent` | `#2E5F8A` (deep slate blue) | Primary CTAs, active states, accent | 56 — your dominant brand color |
| `--color-accent-hover` | `#1A3A5C` (navy) | Hover/pressed | 6 |
| `--color-accent-bright` | `#3A7BD5` (lighter blue) | Highlights, link hover | 15 |
| `--color-accent-secondary` | `#065F46` (emerald) | Secondary accent — success states, secondary CTA differentiation | 7 |
| `--color-logo-grey` | `#7A8190` (warm-cool neutral) | "VENTURES" wordmark + icon mark grey — sampled from logo | NEW (added — required for two-tone wordmark) |
| `--color-error` | `#A8312A` (warm red) | Form errors, destructive — NEW token, not currently in CSS | 0 |

**Why this palette is good:** Deep slate blue + emerald accent is genuinely distinct. It's NOT the typical SaaS bright blue (Bootstrap blue, Tailwind blue-500), and NOT the AI-slop purple/violet. The maritime undertone of `#2E5F8A` reads as "Hudson Bay / coast / serious craft" rather than "every other tech site." The emerald `#065F46` accent gives you a second color to differentiate without competing.

**Contrast budget:** every text/background pair meets WCAG AA (4.5:1 body, 3:1 large headers). Verified pairs:
- Fg `#1A1A2E` on bg `#FFFFFF` = 17.4:1 ✓
- Fg-muted `#5A6478` on bg `#FFFFFF` = 5.7:1 ✓
- Accent `#2E5F8A` on bg `#FFFFFF` = 6.8:1 ✓ (passes AA for body text + UI elements)
- White on accent `#2E5F8A` = 6.8:1 ✓ (button labels readable)
- Accent-hover `#1A3A5C` on white = 11.5:1 ✓
- Accent-secondary `#065F46` on white = 8.6:1 ✓

**Light/dark mode:** spec assumes light mode only for Phase 1. Dark mode is a TODO captured below.

## Brand Mark

The Audax Ventures logo is a circular icon (slate blue `#2E5F8A`) containing a stylized "A" mark in logo-grey `#7A8190`. The "A" reads as a summit/peak — supporting the "audax" (Latin: bold/daring) brand meaning. Wordmark is two-tone uppercase: "AUDAX" in slate blue, "VENTURES" in logo-grey, with generous letter-spacing.

**Usage rules:**
- Header nav: full lockup (icon + wordmark) at 40px height. Use the actual SVG/PNG asset (`/audax-logo.svg`) in production.
- Mobile (<640px): consider icon-only at 32px to save horizontal space (test whether wordmark fits).
- Footer: full lockup, same size as header.
- Favicon: icon-only, square crop.
- Open Graph image: full lockup centered on warm white, ~1200×630.
- Social avatars (LinkedIn, Twitter): icon-only.
- **Never** stretch, recolor, or rearrange. **Never** put the wordmark on a busy background — use the icon-only variant if needed.
- **Never** put a drop shadow or glow on the mark.

The wireframe currently shows a CSS text fallback (two-tone uppercase) because we don't have the SVG asset path locally. Production swap is one line: replace the `<a class="wordmark">` markup with `<a href="/" aria-label="Audax Ventures — home"><img src="/audax-logo.svg" alt="Audax Ventures" height="40" /></a>`.

**Hero block:** 96px top padding from nav. Display copy spans ~70% width on desktop, full-width on mobile. Subhead ~50% width.

**Two-door:** 32px gap between cards. Cards are equal-width, equal-height. Each card is ~512px wide on desktop, full-width stacked on mobile (<768px).

**Card structure:**
- Eyebrow: small caps mono, color `--color-fg-muted`
- Headline: 36px sans, weight 600, color `--color-fg`
- Supporting line: 16px body, color `--color-fg-muted`, max 2 lines
- CTA: arrow link, weight 500, color `--color-fg` → underline expands on hover. NOT a filled button. The card itself is the action — entire card is clickable. Arrow signals direction.

**Card hover (desktop):**
- Border color shifts from `--color-border` to `--color-fg`
- Arrow translates 8px to the right (200ms ease-out)
- Subtle background shift: bg-alt → bg-alt with 2% accent tint
- NO drop shadow lift. Drop shadow lifts are AI slop.

**Card focus (keyboard):** 2px solid outline in `--color-accent`, 4px offset. Visible on every card.

**Click target:** entire card. Accessibility requires `<a>` wrapping the whole card with `aria-label="For founders: I'm building a new product"`.

## Below the Fold (sections 2–4)

### Section 2: Trust Strip
- 8 client logos, monochrome (`--color-fg-muted`), height 32px
- Auto-scroll left → right, 60s loop, pauses on hover
- Eyebrow text above: "Builders who trust us" (small caps mono)
- 64px vertical padding

### Section 3: Metrics Row
Single horizontal row of 3 metrics, equal-width.
```
50+               8–16              98%
PROJECTS SHIPPED  WEEKS TO LAUNCH   ON-TIME DELIVERY
```
Number is 48px Geist Sans 700 in `--color-fg`. Label below is 12px Geist Mono small caps in `--color-fg-muted`. Vertical line dividers between metrics, 1px `--color-border`.

### Section 4: Testimonials
Three testimonials, NOT in a card grid (avoiding AI-slop pattern). Layout:
- Each testimonial gets its own row, full-bleed at 800px max-width centered
- Quote: 24px serif italic (use Geist's italic), 1.4 line height
- Attribution: 14px, name in `--color-fg`, title/company in `--color-fg-muted`
- Vertical separator: 1px line, 96px between testimonials
- NO photos in v1 (until you have permission for headshots; placeholder portrait would weaken trust)

### Final CTA Block
- Single sentence: "Ready to build something Canadian builders are proud of?"
- One button: filled `--color-accent`, white text, "Book a strategy call → Calendly"
- mailto: fallback link below: "Or email hello@audaxventures.ca" — solves the Calendly-outage failure mode flagged in the eng plan

## Mobile (<768px)

- Hero copy: scales to 2.5rem (40px), still left-aligned, full-width
- Two-door: cards STACK vertically, full-width minus 24px page padding, 16px gap between
- Each card slightly shorter (no fixed equal-height)
- Trust strip: still scrolls horizontally; logos 24px height
- Metrics row: stacks to 3 rows, each metric on its own row, separator becomes horizontal
- Testimonials: same pattern, just narrower

**No hamburger menu hidden behind a tap until ≥768px.** Below 640px, nav becomes a hamburger; the two top CTAs ("Founders ▸", "AI for SMEs ▸") become the first two items inside the menu so audience sorting stays one tap away.

---

## Motion

- All transitions: 200ms `ease-out` for hover/focus, 400ms `ease-in-out` for layout shifts.
- Scroll: NONE. No scroll-linked animation, no parallax. Static document. Speed wins over delight on B2B trust pages.
- Reduce motion: `@media (prefers-reduced-motion: reduce)` disables card hover translation and logo carousel auto-scroll.

## Accessibility Spec

| Requirement | Implementation |
|-------------|----------------|
| Single `<h1>` per page | Hero display copy is the H1 |
| Heading hierarchy | h1 → h2 (section heads) → h3 (card headlines) |
| Tab order | Logo → 4 nav links → Book CTA → Card 1 → Card 2 → Section 2 link (none) → Section 4 quotes (none, not interactive) → Final CTA |
| Focus rings | 2px solid `--color-accent`, 4px offset, on every interactive element |
| Touch targets | Minimum 44×44px on mobile (cards far exceed; nav links padded to meet) |
| Alt text | Required on every `<img>`. TypeScript-enforced via wrapper component |
| Reduced motion | Honored for all transitions and the logo carousel |
| Color independence | Hover/focus states use border + arrow movement, not color alone |
| Screen reader card label | Each card `<a>` has explicit `aria-label` describing audience + CTA |

## What This Prevents (AI-slop blacklist passes)

- ✗ Purple/violet gradient — uses white + slate blue + emerald accent
- ✗ 3-column icon-in-circle feature grid — uses two large typographic doors
- ✗ Centered everything — left-aligned hero, left-aligned card content
- ✗ Decorative blobs / wavy SVG dividers — none
- ✗ Emoji as design elements — none
- ✗ Colored left-border on cards — borders are full perimeter, neutral color
- ✗ Generic SaaS hero copy ("Unlock the power of...") — direct, declarative, geographic-anchored
- ⚠ system-ui / default Inter — body uses Inter, but display uses Outfit (the AI-slop concern is *Inter alone* with no display pairing). The Outfit/Inter pairing is a real design choice with character. Keep the existing pairing rather than fighting it.
- ✗ Cookie-cutter section rhythm — sections intentionally varied (poster hero → scrolling logos → typographic metrics → editorial testimonials → final CTA), not 5 identical-height blocks
- ✗ Drop-shadow card lifts on hover — uses border color shift + arrow translate instead

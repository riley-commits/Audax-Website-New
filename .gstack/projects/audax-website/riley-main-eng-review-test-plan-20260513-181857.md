## Key Interactions to Verify

- Two-door picker on `/`: click "I'm building a new product" → lands on `/founders` (URL + page content correct).
- Two-door picker on `/`: click "I'm bringing AI into my business" → lands on `/ai-for-sme`.
- CTA buttons on every page: clicking "Book a Free Strategy Call" opens Calendly embed (or new tab).
- Header nav: every link navigates to the expected route on desktop and mobile.
- Mobile hamburger menu: opens, closes, two-door cards expanded by default.
- Logo carousel on `/`: auto-scrolls without layout shift.
- Service page cross-links: clicking a "related industries" link on a service page navigates correctly.
- Blog post navigation: prev/next or back-to-hub link works.
- Calendly fallback: if Calendly fails to load, `mailto:` fallback link appears (test with network throttling or blocking the Calendly domain).

## Edge Cases

- `/about`: stats section must render real values, NOT zero placeholders. (Existing bug.)
- MDX file with missing required frontmatter field (`description`, `title`, `audience`): build must fail loudly.
- MDX file with invalid `audience` enum value (e.g. `audience: 'random'`): build must fail.
- Pages with very long titles: `<title>` tag truncates gracefully; meta description still 120–160 chars.
- Empty `/blog` (no posts yet): hub page shows a placeholder, doesn't 500.
- Empty `/industries` (no MDX files yet): hub page shows a placeholder, doesn't 500.
- Slow connection: hero image and above-the-fold content render within 2.5s on 3G simulation.
- JavaScript disabled: all content still renders (RSC default), CTAs still work as plain links.
- Keyboard-only navigation: every interactive element reachable via Tab, focusable, with visible focus ring.
- Screen reader: every page has a single `<h1>`, semantic heading order, alt text on every image.

## Critical Paths

- **Homepage → audience door → Calendly booking.** The single most important funnel.
  - `/` loads → two-door picker visible above the fold → click → audience page renders → CTA visible → Calendly opens.
- **Google search → service page → audience door → Calendly booking.** SEO long-tail path.
  - Direct-load `/services/ai-consulting` → page renders with correct SEO meta → "talk to us" CTA → routes to `/ai-for-sme` or directly opens Calendly.
- **Google search → industry page → audience door → Calendly booking.** Industry long-tail path.
  - Direct-load `/industries/manufacturing` → renders → CTA → Calendly.
- **`/about` → trust validation → CTA.** Credibility path.
  - Direct-load `/about` → team photos + real stats (not zeros) + mission → CTA visible.
- **Sitemap discoverability.**
  - `/sitemap.xml` loads → contains every dynamic route → Google can index everything.

## Accessibility Verification

- axe-core scan on `/`, `/founders`, `/ai-for-sme`, `/about` — zero violations.
- Keyboard nav: every CTA, nav link, and two-door card reachable in tab order.
- Color contrast: WCAG AA on all text.
- Alt text: every `<img>` has non-empty alt OR is decorative (`role="presentation"`).

## SEO Verification

- Every page has unique `<title>` (no duplicates).
- Every page has `<meta name="description">` 120–160 chars.
- Every page has correct `canonical` URL.
- Open Graph image renders correctly when shared (test with og-debugger / Twitter card validator).
- JSON-LD structured data validates against schema.org (Organization on `/`, Service on service pages, Article on blog).
